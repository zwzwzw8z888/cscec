# 数据表格写作指南

> 本指南面向中建四局公文中的数据分析场景，规范表格设计与数据阐述方式。

## 一、基本原则

1. **数据驱动结论**：每项数据须配有分析性文字，避免罗列数据而无解读
2. **先总后分**：段落先给结论或判断，再用表格和数据支撑
3. **对比先行**：与历史同期、计划目标、行业基准进行对比
4. **表格精简**：一张表格聚焦一个分析维度，避免信息过载

## 二、表格结构规范

### 2.1 基本要求

| 属性 | 要求 |
|------|------|
| 字体 | 仿宋_GB2312（中文），Times New Roman（数字） |
| 字号 | 小四（12pt），标题行加粗 |
| 行距 | 单倍行距 |
| 行高 | 不低于 7mm |
| 对齐 | 整体适应窗口宽度 |
| 边框 | 单线边框，颜色自动 |
| 标题行 | 居中，加粗 |
| 数据行 | 数值右对齐，文字左对齐或居中 |

### 2.2 标题命名

表格上方设置表序和表名，格式：

- **单表**：表1 XXXX
- **多表**：表1 XXXX / 表2 XXXX（连续编号）

标题字体与正文一致（仿宋_GB2312，三号），居中对齐。

## 四、表格后的分析写作模式

### 4.1 "结论-数据-原因"三段式

每张表格后采用以下写作结构：

**第一段：核心结论（1-2句）**
> 从表1可见，2024年一季度平台整体使用量呈上升趋势，累计活跃用户369人，数据处理量达7,020万条，同比分别增长26.1%和15.8%，平台推广效果显著。

**第二段：关键数据解读（2-3句）**
> 其中，工程部以3,120万条数据量位居首位，同比增长28.6%，主要得益于BIM数据集成模块的上线。数字化部活跃用户数最多（156人），反映了总部数字化意识的持续增强。

**第三段：问题与建议（1-2句，可选）**
> 财务部使用量增速相对偏低（+12.8%），建议加强业务培训并优化财务数据填报流程。

### 4.2 对比分析写作模板

> 如表2所示，本期各项关键指标均实现同比增长。其中接口调用量达128.5万次，超出年度月均目标（10万次）的7.1%，系统可用率稳定在99.8%以上。需关注平台日活（1,250人）距年度目标（1,500人）仍有16.7%的差距，建议二季度加大推广力度。

### 4.3 趋势判断写作模板

> 综合表1-3数据分析，DMP平台呈现以下发展趋势：一是用户规模稳步扩大，月均增速保持在15%以上；二是数据质量持续改善，异常数据率从上季度的3.2%降至1.8%；三是系统运行平稳，可用率维持在99.5%以上。预计二季度平台日活将突破1,500人目标。

## 五、表格数据呈现规范

### 5.1 数值格式

- 数值使用千分位分隔符：1,234.56
- 百分比保留一位小数：12.5%
- 同比变化使用 + / - 前缀：+15.3% / -2.1%
- 大数量级可使用万/亿单位：2,340万、15.6亿

### 5.2 状态标记

- 已完成：✅ 或"已完成"
- 进行中：🔄 或"进行中"
- 待启动：⏳ 或"待启动"
- 延期：⚠️ 或"延期"
- 风险等级：高/中/低（可配颜色提示）

### 5.3 合计行

- 数据类表格底部设置合计行
- 合计行加粗或使用浅色底纹区分
- 无合计意义的列用"—"或空白标注

## 六、docx-js 表格实现要点

### 6.1 表格宽度计算

A4纸张有效内容宽度 = 页面宽度 - 左边距 - 右边距
= 11906 - 1588 - 1474 = 8844 DXA

```javascript
const CONTENT_WIDTH = 8844; // A4有效内容宽度(DXA)
const border = { style: BorderStyle.SINGLE, size: 1, color: "000000" };
const borders = { top: border, bottom: border, left: border, right: border };
```

### 6.2 标题行单元格

```javascript
new TableCell({
  borders,
  width: { size: 1500, type: WidthType.DXA },
  shading: { fill: "F0F0F0", type: ShadingType.CLEAR },
  margins: { top: 60, bottom: 60, left: 100, right: 100 },
  verticalAlign: VerticalAlign.CENTER,
    children: [new Paragraph({
    alignment: AlignmentType.CENTER,
    children: [new TextRun({ text: "列标题", bold: true, font: "Times New Roman", eastAsia: "仿宋_GB2312", size: 24 })]
  })]
})
```

### 6.3 数据行单元格

```javascript
new TableCell({
  borders,
  width: { size: 1500, type: WidthType.DXA },
  margins: { top: 40, bottom: 40, left: 100, right: 100 },
  verticalAlign: VerticalAlign.CENTER,
    children: [new Paragraph({
    alignment: AlignmentType.RIGHT, // 数值右对齐
    children: [new TextRun({ text: "1,234.56", font: "Times New Roman", eastAsia: "仿宋_GB2312", size: 24 })]
  })]
})
```

### 6.4 关键注意事项

- 所有 `columnWidths` 数组之和必须等于表格 `width`
- 每个单元格的 `width` 必须与对应 `columnWidth` 一致
- 使用 `ShadingType.CLEAR`（非 SOLID）防止黑色背景
- 必须设置单元格 `margins` 保证内边距
- 数据列使用 `AlignmentType.RIGHT` 右对齐
