# 一键出通报工具 — 设计规格

## 概述

为"中建四局DMP推广应用情况月度通报"提供一个一键生成工具。使用者每月将系统导出的固定结构 Excel 丢入输入文件夹，双击运行脚本，自动生成完整 Word 通报文档（含红头、标题、正文分析文字、数据表格、落款），并通过 SQLite 保存本月数据，供下月自动环比对比。

## 运行方式

1. 月初将本月导出的几个 Excel 文件放入 `数据输入/`
2. 双击 `一键出通报.bat`
3. 按提示输入月份（如 `4`）
4. Word 通报自动生成到 `通报输出/`，文件名如 `关于中建四局DMP推广应用情况的通报（2026年4月）.docx`

## SQLite 数据存储

核心表 `snapshots`，按字段名纵向存储（key-value 风格）：

| 字段 | 说明 |
|------|------|
| year_month | 如 '2026-04' |
| section | 如 '新开工项目上线' |
| unit_name | 如 '一公司' 或 '合计' |
| metric_kv | JSON 字符串，存 {项目数量:5, 已上线数量:3, 比例:60%} |

环比逻辑：读取 Excel 算出本月值 → 查 `WHERE year_month = 上个月` → 填入"截止上月"列。

## 配方（Recipe）规范

每个 recipe 文件定义一个 `process(dataframes)` 函数：

- 入参：`dataframes` dict，key 为 Excel 文件名，value 为 pandas DataFrame（所有 sheet）
- 出参：dict，包含 `section_name`、`table_data`（二维列表）、`pre_text`（表格前引导文字）、`post_note`（表格后注释）

示例伪码：
```
def process(dataframes):
    df = dataframes['项目基本信息台账.xlsx']['项目信息台账']
    df_本月 = df[df['系统应用阶段'] == '已上线']
    分组 = df_本月.groupby('所属末级管理责任主体').agg(...)
    return {'section_name':'新开工项目上线', 'table_data':[...], ...}
```

## 通报文档结构（按三月PDF）

1. 红头 + 发文字号 + 标题 + 主送单位
2. 一、建设情况（需求解决 + 问题处理 + 新增需求表）
3. 二、推广应用情况
   - （一）体系建设（纯文字段落）
   - （二）新开工项目当月上线
   - （三）新开项目业财集成
   - （四）增量项目业财集成
   - （五）本月新增单据量（市场/合同/经济/生产/供应链5个子表）
   - （六）筑共赢商务协同
   - （七）新开项目供应链集成
4. 三、系统运维情况
   - （一）各单位运维
   - （二）各业务板块运维
5. 落款（中建四局流程与数字化管理部 + 日期）
6. 抄送

## 格式规范

复用现有 `中建四局公文写作` 技能常量：
- 红头：方正小标宋简体，初号(42pt)，红色
- 标题：方正小标宋简体，二号(22pt)，居中
- 一级标题：黑体，三号(16pt)
- 二级标题：楷体_GB2312，三号(16pt)
- 正文：仿宋_GB2312，三号(16pt)，首行缩进2字符
- 表格内容：仿宋_GB2312，小四(12pt)，标题行加粗
- 行距：固定值 28.9磅 = 579 twips
- 纸张 A4，页边距上37/下35/左28/右26mm

## 首期运行处理

首期使用（如 3 月）时，SQLite 中无上月数据。工具提示"未找到上月数据，截止上月列留空"，表格中对应列显示 "/"，用户手动补充后，自动存档本月数据。次月起全自动。

## 技术栈

- Python 3（openpyxl / pandas 读取 Excel，python-docx 生成 Word）
- SQLite3（标准库，零依赖）
- 文件编码 UTF-8

## 约束

- 仅支持 .xlsx 格式输入
- 系统导出 Excel 结构需保持稳定（列头变动的概率很低，真变动了改对应配方即可）
- 输出 .docx，如需继续微调可在 Word 中手工修改
