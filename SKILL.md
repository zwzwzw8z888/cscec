# 中建四局公文写作专家

## 概述

本技能专精于按照中建四局公文基本格式（2023.9.27版）撰写专业工作文档，确保文档格式合规、结构清晰、语言规范。适用于请示、报告、批复、通知、函件、会议纪要等常见公文类型，以及数据分析报告、工作总结等专业文档。

详细格式规范参见 `references/format-spec.md`，数据表格写作指南参见 `references/data-table-guide.md`。

## 写作流程

### 第一步：确认文档类型

根据用户需求确定文种：

| 文种 | 适用场景 | 固定结尾 |
|------|----------|----------|
| 请示 | 向上级请求批准事项 | 妥否，请示 |
| 报告 | 向上级汇报工作情况 | 特此报告 |
| 通知 | 向下级传达事项 | 无固定结尾 |
| 批复 | 答复下级请示 | 特此批复 |
| 函件 | 平级或不相隶属机关之间 | 无固定结尾 |

### 第二步：确定文档结构

根据文档类型选择对应结构骨架：

**标准公文结构（请示/报告/通知/批复）：**
1. 发文机关标志（红头）—— 方正小标宋简体，初号，红色
2. 发文字号 —— 仿宋_GB2312，三号
3. 签发人（上行文必填）
4. 标题 —— 方正小标宋简体，二号，居中
5. 主送单位（抬头）
6. 正文（含分级标题）
7. 结束语
8. 附件说明（如有）
9. 落款与日期
10. 版记（抄送、印发单位及日期）

**数据分析报告结构：**
1. 标题 —— 概括报告核心主题
2. 摘要 —— 一段话说明背景、目的、主要结论
3. 正文
   - 背景与目的
   - 数据来源与分析方法
   - 分析结果（**必须使用表格**呈现关键数据）
   - 问题与风险
   - 建议与措施
4. 结语
5. 附件说明（如有）

### 第三步：执行内容生成

使用 docx 技能（docx-js）生成文档时，严格遵循格式规范：

#### 字体映射表

| 元素 | 中文字体 | 英文/数字字体 | 字号 | 样式 |
|------|----------|--------------|------|------|
| 发文机关标志（红头） | 方正小标宋简体 | Times New Roman | 初号(42pt) | 红色，不加粗 |
| 标题 | 方正小标宋简体 | Times New Roman | 二号(22pt) | 不加粗，居中 |
| 一级标题 | 黑体 | Times New Roman | 三号(16pt) | 不加粗 |
| 二级标题 | 楷体_GB2312 | Times New Roman | 三号(16pt) | 可加粗 |
| 三级标题 | 仿宋_GB2312 | Times New Roman | 三号(16pt) | 可加粗 |
| 四级标题 | 仿宋_GB2312 | Times New Roman | 三号(16pt) | 可加粗 |
| 五级标题 | 仿宋_GB2312 | Times New Roman | 三号(16pt) | 可加粗 |
| 正文内容 | 仿宋_GB2312 | Times New Roman | 三号(16pt) | 不加粗 |
| 表格内容 | 仿宋_GB2312 | Times New Roman | 小四(12pt) | 标题行加粗 |

#### 字号速查（磅值/DXA）

| 字号名称 | 磅值 | docx-js half-points |
|----------|------|---------------------|
| 初号 | 42pt | 84 |
| 小初 | 36pt | 72 |
| 一号 | 26pt | 52 |
| 小一 | 24pt | 48 |
| 二号 | 22pt | 44 |
| 小二 | 18pt | 36 |
| 三号 | 16pt | 32 |
| 小三 | 15pt | 30 |
| 四号 | 14pt | 28 |
| 小四 | 12pt | 24 |

#### 页面设置

- 纸张：A4（11906 × 16838 DXA）
- 页边距：上 37mm（≈2094 DXA），下 35mm（≈1984 DXA），左 28mm（≈1588 DXA），右 26mm（≈1474 DXA）
- 行距：固定值 28.9磅（≈578 twips = 579 DXA），全文保持一致
- 双面打印，左侧装订

#### 分级标题格式

```
一、一级标题（黑体，三号，首行缩进2字符）
（一）二级标题（楷体_GB2312，三号，首行缩进2字符）
1. 三级标题（仿宋_GB2312，三号，首行缩进2字符）
（1）四级标题（仿宋_GB2312，三号，首行缩进2字符）
① 五级标题（仿宋_GB2312，三号，首行缩进2字符）
```

#### 正文段落规则

- 字体：仿宋_GB2312（中文），**数字使用 Times New Roman**
- 字号：三号（16pt）
- 对齐：两端对齐（justify）
- 首行缩进：2字符
- 行距：固定值 28.9磅

### 第四步：表格规范

数据分析部分必须结合表格进行内容阐述，遵循以下规范：

1. **表格字体**：仿宋_GB2312 小四号（12pt），标题行加粗
2. **行距**：单倍行距
3. **行高**：不低于 7mm
4. **整体适应窗口**
5. **表格前后**：必须有引导文字说明和总结分析

详细表格设计模式参见 `references/data-table-guide.md`。

### 第五步：格式校验

文档生成后，逐项检查：

- [ ] 页面大小 A4，页边距正确
- [ ] 发文机关标志（如需要）：方正小标宋简体，初号，红色，居中
- [ ] 标题：方正小标宋简体，二号，不加粗，居中
- [ ] 主送单位：仿宋_GB2312，三号，无缩进
- [ ] 各级标题字体正确（黑体/楷体/仿宋）
- [ ] 正文：仿宋_GB2312，三号，首行缩进2字符
- [ ] 数字字体：Times New Roman
- [ ] 行距全文一致（28.9磅固定值）
- [ ] 结束语符合文种规范
- [ ] 落款日期格式正确（202X年X月X日）
- [ ] 数据表格标题行加粗，内容为小四号
- [ ] 附件说明格式正确

## 写作语言规范

### 用词要求

- 使用正式书面语，避免口语化表达
- 不得使用网络用语、流行语或俚语
- 数量词使用规范：用"一"不用"1"（如"一项措施"不用"1项措施"）
- 日期使用阿拉伯数字，年份标全称
- 百分号前加数字（如"同比增长12.5%"）

### 句式要求

- 语句简洁有力，避免冗长复杂句
- 多用短句，少用长句
- 主动语态为主，避免被动语态堆砌
- 段落控制在 200 字以内

### 数据分析写作要点

- **先总后分**：每段先给结论，再用数据支撑
- **表格先行**：关键数据必须以表格形式呈现
- **文字解读**：表格后必须有分析性文字说明数据意义
- **对比分析**：与上期/同期/目标进行对比
- **趋势判断**：基于数据给出趋势性判断

## 资源文件

### 核心代码文件

| 文件 | 职责 | 这决定放哪里 |
|------|------|-------------|
| `formatter_core.py` | 主入口，流程编排 | 修改整体流程时改这里 |
| `constants.py` | 格式常量 + ⭐全局规则集中声明 | 新增全局判断函数（is_xxx/should_xxx）加这里 |
| `detect.py` | 段落层级检测 + 标题判断 + HeadingCounter | 修改"什么算标题"逻辑时改这里 |
| `check.py` | 7 个审查检测函数 | 新增格式问题检测规则时加这里 |
| `comment.py` | 批注写入 + 页码 | 修改批注显示方式时改这里 |
| `table.py` | 表格提取/重建（底色/字体颜色保留） | 修改表格处理规则时改这里 |
| `run_format.py` | CLI 命令行入口（不变） | 不存规则，只负责传参调用 |

**架构原则**：`constants.py` 是规则的"宪法"，所有跨模块的判断函数统一在此声明，其他模块只调用不复制。

**使用方法**：
```bash
# 基本用法
python run_format.py <输入文件.docx> [输出文件.docx]

# 示例
python run_format.py "报告草稿.docx" "报告-格式化.docx"
```

**代码架构**：
- 调整格式常量 → 编辑 `constants.py`
- 新增全局规则判断 → 编辑 `constants.py`（函数声明），在相应模块中调用
- 修改层级检测 → 编辑 `detect.py`
- 新增审查规则 → 编辑 `check.py`
- 修改批注显示 → 编辑 `comment.py`
- 修改表格处理 → 编辑 `table.py`
- 修改整体流程 → 编辑 `formatter_core.py`
- 修改输入输出逻辑 → 编辑 `run_format.py`
- 所有文件均在技能文件夹内，随技能一起分发

## docx-js 实现注意事项

### 字体兼容性

docx-js 中设置中文字体时，需同时设置 `eastAsia` 属性：

```javascript
run: {
  font: "Calibri",
  eastAsia: "仿宋_GB2312", // 关键：中文回退字体
  size: 32 // 三号 = 16pt = 32 half-points
}
```

### 行距设置

**⚠️ 关键警告**：行距单位是 twips（1磅=20twips），28.9磅 = 579 twips。**绝对不能传入磅值或像素值作为行距参数**，否则会导致行距变为数万磅，文档打开显示空白。

```javascript
paragraph: {
  spacing: {
    line: 579, // 固定值 28.9磅 = 579 twips，不是 28.9 也不是 2890！
    lineRule: "exact" // 使用 exact 而非 auto/atLeast
  }
}
```

### 首行缩进

```javascript
paragraph: {
  indent: {
    firstLine: 640 // 2字符 × 320 DXA/字符（三号字）
  }
}
```

### 红头文字

```javascript
new Paragraph({
  alignment: AlignmentType.CENTER,
  spacing: { line: 900, lineRule: "exact" }, // 45磅行距
  children: [
    new TextRun({
      text: "中国建筑第四工程局有限公司文件",
      font: "方正小标宋简体",
      eastAsia: "方正小标宋简体",
      size: 84, // 初号 = 42pt
      color: "FF0000", // 红色
    })
  ]
})
```
